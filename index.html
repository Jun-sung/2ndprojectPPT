企画意図 (Concept)

PURPOSE
開発目的

Spring Boot + MyBatis + Spring Security を活用したフルスタック開発の一連の流れ （認証・DB・外部API連携・フロント）を通して、 実務に近い形でのバックエンド開発スキルを身につけることを目的としました。
国土交通省 Reinfolib API と連携し、実データを扱いながら データ取得 → 加工 → 地図表示 まで実装しました。

PROBLEM
解決したい課題

日本では不動産情報とハザードリスク情報が別々のサービスに分かれており、 「価格は安いが災害リスクは高い地域」などを一目で判断しづらいという課題があります。
SAFE HOME では、 「物件情報」と「災害リスク」を一画面で統合確認 できる UI を提供し、安全な住まい探しをサポートすることを目指しました。

FEATURES
主要機能
  • 物件情報の可視化： 国土交通省の不動産取引価格データを Leaflet 地図上にポップアップ付きで表示
  • ハザードマップ統合： 洪水（6段階）・津波（5段階）・土砂災害（2段階）のリスクゾーンを GeoJSON ポリゴンとして重ね合わせ
  • AIチャット相談： Gemini 1.5 Flash API を用いて、地域の特徴や条件に合うエリアの候補を日・韓 2言語で提案

07
Troubleshooting
07 / 15
ISSUE 01

Data Overflow

MySQL データ長超過

タイルキー（region_code）が DB カラムの文字数制限を超えてデータが切り捨てられる現象が発生。

SOLUTION

Prefix Mapping 戦略を導入

繰り返す文字列を短いコードに変換（FLOOD→FL 等）し、キー長を 60% 以上短縮。

ISSUE 02

State Loss

言語切替時に検索パラメータが消える

日/韓 言語切替の際、現在表示中の地図座標と検索フィルターのパラメータが初期化される問題が発生。

SOLUTION

Dynamic Link Resolver を実装

URLSearchParams で現在のクエリ文字列を解析し、言語切替ボタンのリンクに動的に結合するスクリプトを適用。

ISSUE 03

Race Condition

非同期レスポンスの逆転現象

素早いフィルター操作時、古いリクエストのレスポンスが最新の結果を上書きし、誤った情報が表示されていた。

SOLUTION

Request ID バリデーション

AbortController と固有リクエスト ID 体系を導入し、最後に呼び出されたリクエストのレスポンスのみをレンダリング。

DB 設計の重要性

カラム長・型設計のミスは深刻なデータ破損を引き起こす。VARCHAR の上限設計は API のキー仕様を事前調査した上で決定すべきと学んだ。

URL 状態管理の設計

UI 上の状態（検索条件・言語）は URL に反映させることで、リロードや遷移後も再現できる設計が UX に直結すると実感。

非同期処理のリスク管理

非同期通信は「速さ」だけでなく「順序の保証」が必要。AbortController でリクエストをキャンセルする手法はフロント開発の基礎として定着した。
